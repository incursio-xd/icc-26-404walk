# FogGuard — Edge AI Wearable for Freezing of Gait Detection

FogGuard is an ankle-worn wearable device that detects and predicts Freezing of Gait (FoG) episodes in Parkinson's disease patients using on-device TinyML inference. It delivers real-time haptic and audio cueing to interrupt freeze episodes, and sends caregiver alerts over BLE when a full freeze is detected.

---

## System Architecture

```
ICM-42688-P IMU (100 Hz)
        ↓ I2C + LDMA
EFR32xG26 SoC
  ├── Sliding Window (300 samples, 50% overlap)
  ├── TFLite Micro Inference (1D CNN, INT8, MVP accelerated)
  ├── Gait FSM (Normal / Pre-Freeze / Freeze)
  ├── Haptic + Audio Feedback (on-device, zero BLE dependency)
  └── BLE GATT Notifications
        ↓ BLE
Raspberry Pi 4 (BLE-WiFi bridge)
        ↓ WiFi / LAN
Node.js Backend + MongoDB
        ↓ Socket.io
React Dashboard (real-time caregiver view)
```

---

## Hardware Requirements

| Component | Part | Purpose |
|---|---|---|
| Microcontroller | EFR32MG26B510F3200IM68 (BRD2608A) | Main SoC — inference, BLE, peripherals |
| IMU | ICM-42688-P (TDK InvenSense) | 6-axis accelerometer + gyroscope |
| Haptic Motor | LRA coin motor | Vibrotactile cueing |
| Buzzer | 5V passive piezo | Auditory cueing |
| Battery | LiPo 3.7V 500–1000mAh | Power supply |
| Gateway | Raspberry Pi 4 | BLE-to-WiFi bridge |

## Hardware Setup

```
BRD2608A Dev Kit
├── PC04 (Pin 5) → ICM-42688-P SCL
├── PC05 (Pin 6) → ICM-42688-P SDA
├── 3.3V → ICM-42688-P VDD
├── GND  → ICM-42688-P GND
├── TBD  → LRA haptic motor (PWM via TIMER, Week 9-10)
└── TBD  → Passive buzzer (PWM via TIMER, Week 9-10)

ICM-42688-P
├── I2C Address: 0x68 (AD0 pin low)
├── WHO_AM_I: 0x47
├── Config: ±4g accel, ±500dps gyro, 100Hz ODR
└── Data register: 0x1F (12-byte burst, all 6 axes)
```

## Build Environment Setup

### Windows

1. Install [Simplicity Studio v6](https://www.silabs.com/developers/simplicity-studio)
2. Inside SSv6, install via **Packages**:
   - Simplicity SDK Suite v2025.12.3
   - Technology: **Bluetooth**, **32-bit MCU**, **AI/ML**
3. Install [VS Code](https://code.visualstudio.com/) and the Simplicity Studio VS Code Extension (from SSv6 → Tools)
4. Add the GNU ARM toolchain to your system PATH:
   ```
   C:\Users\<YOU>\.silabs\slt\installs\conan\p\gcc-<hash>\p\bin
   C:\Users\<YOU>\.silabs\slt\installs\conan\p\ninja<hash>\p
   C:\Users\<YOU>\.silabs\slt\installs\conan\p\cmake<hash>\p\bin
   ```
   (Exact paths shown in `cmake_gcc/toolchain.cmake` after project generation)
5. Open `embedded/firmware/` as the working project in SSv6
6. To build, run from `embedded/firmware/`:
   ```bat
   build.bat
   ```
   This auto-patches the cmake file and runs a full build.

### Linux / macOS

Not supported for this project — Simplicity Studio v6 is Windows/macOS only. Linux builds require manual GNU ARM toolchain setup and are not tested.

## Flashing

After a successful build, flash using Simplicity Commander or the SSv6 flash button:

```
Commander flash cmake_gcc/build/base/icc-26-fogguard.hex
```

Or via J-Link directly from VS Code using the Simplicity Studio extension.

## Debug Environment

### RTT Logging (recommended)
RTT (Real-Time Transfer) is the primary debug channel — no extra wires needed.

1. Flash the firmware
2. Open **Simplicity Studio → Tools → J-Link RTT Viewer**
3. Connect to the device — all `app_log()` output appears here in real time

You will see:
```
=== FogGuard Starting ===
[IMU] OK
[BLE] Advertising as FogGuard
[IMU] AX: 0.012 AY:-0.003 AZ: 1.001 | GX: 0.120 GY:-0.031 GZ: 0.008
```

### UART / VCOM (alternative)
Connect via USB virtual COM port at **115200 baud** using any serial monitor (PuTTY, Tera Term, VS Code Serial Monitor extension).
